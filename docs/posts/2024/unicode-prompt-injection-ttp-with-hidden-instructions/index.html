Novel Unicode Prompt Injection TTP Allows Smuggling Instructions

Posted on Jan 13, 2024
#aiml #machine learning #threats #ai injections #llm #ttp

A few days ago Riley Goodside posted about a tricky discovery he made “LLM prompt injection via invisible instrucitons in pasted text”. It allows inserting hidden text inside a string using a special set of Unicode characters from the [Tags"Unicode Block](https://en.wikipedia.org/wiki/Tags_(Unicode_block).

The cool part: The “Tag” Unicode block mirrors ASCII and it is not rendered These characters remain hidden in user interfaces (if one ommits the flag begin characters first) hence they stay unnoticable for users, but LLMs interpret them as their ASCII eqivivalents.

The implications –> Attackers can smuggling ASCII text into LLM prompts that perform unwanted or malicious instructions.

Conclusion

This is an important new attack avenue for LLMs (and maybe even beyond) and it seems a good idea to filter out Tag Unicode characters at prompting and response layers.

Appendix

Here are two tools that can be used for testing and building custom payloads.

  • Python script by rez0:
import pyperclip

def convert_to_tag_chars(input_string):
     return ''.join(chr(0xE0000 + ord(ch)) for ch in input_string)

# Example usage:
user_input = input("Enter a string to convert to tag characters: ")
tagged_output = convert_to_tag_chars(user_input)
print("Tagged output:", tagged_output)
pyperclip.copy(tagged_output)
  • Golang tool (llm-ascii-smuggler-) doing pretty much the same as above Pythn script. But, I like Joseph’s Python version better because it directly copies the result in the clipboard!

	input := os.Args[1]
	output := ""
